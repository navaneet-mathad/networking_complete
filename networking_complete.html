// SECTION 02 · INTERNET MODEL
TCP/IP Model

The TCP/IP model (also called the Internet model or DoD model) is the practical implementation used on the Internet. It has 4 or 5 layers and maps to the OSI model.

TCP/IP LAYERS (5-layer)

5. Application
HTTP, FTP, SMTP, DNS, DHCP, Telnet, SSH. Combines OSI layers 5, 6, 7. Provides network services directly to user applications.
4. Transport
TCP (reliable, connection-oriented) and UDP (fast, connectionless). Handles port numbers, segmentation, and end-to-end delivery.
3. Internet / Network
IP (v4/v6), ICMP, IGMP. Logical addressing, routing between networks, packet forwarding. IP is the core protocol here.
2. Data Link
Ethernet, Wi-Fi (802.11), PPP, ARP. Frames, MAC addresses, LLC sublayer, error detection (CRC).
1. Physical
Actual transmission of bits: cables (Cat5e/6/7), fiber optics, radio waves, voltage levels, pin layouts.

// SECTION 03A · TCP MECHANICS
TCP Three-Way Handshake

TCP establishes a connection before data transfer using a three-way handshake. This ensures both parties are ready and synchronizes sequence numbers.

1
SYN — Client → Server SYN=1SEQ=x
Client sends a SYN (synchronize) packet with an initial sequence number. Requests to establish a connection.
2
SYN-ACK — Server → Client SYN=1ACK=x+1
Server acknowledges the SYN, sends its own SYN. ACK = client's SEQ+1. SEQ = y (server's ISN).
3
ACK — Client → Server ACK=y+1
Client acknowledges server's SYN. Connection established. Data transfer begins.
4
FIN-ACK — Connection Teardown FIN=1
4-way termination: FIN → ACK → FIN → ACK. Half-close allows one side to finish while the other continues sending.

// SECTION 05 · PATH SELECTION
Routing Protocols

Routing protocols determine the best path for packets across networks. Classified as Interior Gateway Protocols (IGP) within an AS, or Exterior Gateway Protocols (EGP) between ASes.

🔄
RIP
DISTANCE VECTOR · IGP
Routing Information Protocol. Selects path with fewest hops (max 15). Sends full routing table every 30 seconds. Simple but slow convergence.
Algorithm: Bellman-Ford · Admin Dist: 120
🗺️
OSPF
LINK STATE · IGP
Open Shortest Path First. Each router has full network topology. Fast convergence, supports VLSM, areas reduce overhead. Industry standard IGP.
Algorithm: Dijkstra (SPF) · Admin Dist: 110
⚡
EIGRP
ADVANCED DISTANCE VECTOR · IGP
Enhanced Interior Gateway Routing Protocol (Cisco). Uses bandwidth + delay metric. Partial updates, rapid convergence. Hybrid of distance-vector and link-state.
Algorithm: DUAL · Admin Dist: 90/170
🌐
BGP
PATH VECTOR · EGP
Border Gateway Protocol — the protocol of the Internet. Routes between Autonomous Systems. Uses TCP port 179. Policy-based routing, not just shortest path.
Algorithm: Path Vector · Admin Dist: 20/200
📋
IS-IS
LINK STATE · IGP
Intermediate System to Intermediate System. Similar to OSPF but runs directly on L2. Popular in ISP core networks. Supports large networks efficiently.
Algorithm: Dijkstra · Admin Dist: 115
🔧
Static / Default
MANUAL · ALL
Manually configured routes. Default route (0.0.0.0/0) is the "gateway of last resort." Reliable but doesn't adapt to topology changes.
Admin Dist: 1 (static) / 0 (connected)

ROUTING CONCEPTS
Longest Prefix Match: Router picks the most specific route (longest mask) in the table.
AS (Autonomous System): Network under single administrative control. Has unique ASN (1–65535 public).
Convergence: Time for all routers to agree on the same routing table after a topology change.
Split Horizon: Don't advertise a route back out the same interface it was learned from. Prevents routing loops.
// SECTION 06 · NETWORK LAYOUT
Network Topologies

The physical or logical arrangement of nodes and connections in a network. Each topology has trade-offs in cost, performance, and fault tolerance.

Bus
All nodes share one backbone cable. Simple and cheap.
+ Low cost + Easy to extend
− Single point of failure − Collisions
Star
All nodes connect to a central hub or switch. Most common LAN topology.
+ Easy troubleshoot + Centralized control
− Hub = single point of failure
Ring
Nodes form a closed loop. Token passing controls access.
+ Equal access + No collisions
− Any node failure breaks ring
Mesh
Every node connects to every other. Used in WANs and Internet.
+ Highly redundant + Fault tolerant
− Expensive + complex wiring
Tree
Hierarchical star topology. Root → branches → leaves. Used in enterprise.
+ Scalable + Organized
− Root failure = full outage

// SECTION 09 · LAYER 2
Switching & VLANs

Ethernet switching techniques, VLAN segmentation, and Spanning Tree Protocol for loop prevention.

SWITCHING METHODS

How a switch decides to forward a frame:

  • Store-and-Forward: Receives full frame, checks CRC, then forwards. Most reliable, highest latency.
  • Cut-Through: Forwards after reading destination MAC only. Low latency, no error checking.
  • Fragment-Free: Reads first 64 bytes (collision window) then forwards. Compromise.

VLANs (802.1Q)

Virtual LANs logically segment a network regardless of physical location:

  • VLAN tag: 4-byte 802.1Q tag inserted in Ethernet frame
  • Tag contains: TPID (0x8100), PCP, DEI, VID (12-bit = 4094 VLANs)
  • Access ports: untagged, one VLAN
  • Trunk ports: tagged, carry multiple VLANs
  • Native VLAN: untagged traffic on trunk
  • Inter-VLAN routing needs L3 switch or router-on-a-stick

STP (802.1D)

Spanning Tree Protocol prevents broadcast storms by blocking redundant paths:

  • Root Bridge election: lowest Bridge ID (priority + MAC)
  • Root Port: best path to root bridge
  • Designated Port: best port per segment
  • Port States: Blocking → Listening → Learning → Forwarding
  • RSTP (802.1w): rapid convergence (<1 sec vs 30-50 sec)
  • MSTP (802.1s): multiple spanning trees for VLANs

// SECTION 11 · NETWORK SECURITY
Network Security

Common attacks, defenses, and security protocols that protect network infrastructure and data in transit.

🔒 TLS/SSL

Transport Layer Security encrypts data between client and server. TLS 1.3 is current standard. Uses asymmetric crypto (RSA/ECC) for key exchange, symmetric (AES) for data. X.509 certificates verify identity.

TLS 1.3AES-256ECDHEPort 443

🔴 DDoS Attack

Distributed Denial-of-Service floods a target with traffic from many sources. Types: volumetric (UDP flood), protocol (SYN flood), application (HTTP GET flood). Mitigated by rate limiting, blackholing, CDN.

SYN FloodAmplificationBotnet

🎭 ARP Spoofing

Attacker sends fake ARP replies to associate their MAC with a legitimate IP. Enables Man-in-the-Middle attacks. Mitigated by Dynamic ARP Inspection (DAI) and static ARP entries.

MitMPoisoningDAI

🕵️ Packet Sniffing

Capturing network traffic in promiscuous mode. Tools: Wireshark, tcpdump. Passive eavesdropping on unencrypted traffic. Defense: use encryption (TLS, SSH), network segmentation, switch security.

WiresharkPromiscuousPassive

🔐 IPSec

Internet Protocol Security. Works at Layer 3. Two modes: Transport (encrypts payload only) and Tunnel (encrypts entire packet — used in VPNs). Protocols: AH (authentication) and ESP (encryption + auth).

AHESPIKEv2VPN

🔑 Network Access Control

802.1X port-based authentication using RADIUS/TACACS+. EAP frameworks verify identity before granting network access. Separates authenticated from unauthenticated traffic at the switch level.

802.1XRADIUSEAPTACACS+

Static NAT
1-to-1 mapping. One private IP ↔ one public IP. Used for servers needing consistent external IP.

// SECTION 16 · TCP MECHANICS
Flow Control & Congestion

TCP mechanisms that prevent overwhelming receivers and network links, ensuring reliable and fair data delivery.

FLOW CONTROL

Prevents a fast sender from overwhelming a slow receiver. Uses the receive window (rwnd) advertised in TCP header.

1
Sliding Window: sender can have multiple unacknowledged segments in flight (up to rwnd)
2
Receiver advertises window size. When buffer fills, sets window=0 (pause)
3
Zero Window Probe: sender periodically checks if receiver is ready
4
Window scales up as receiver processes data and frees buffer space

CONGESTION CONTROL

Prevents network collapse from too much data. Uses congestion window (cwnd) separate from rwnd.

Slow Start
cwnd starts at 1 MSS, doubles every RTT until ssthresh. Exponential growth phase.
Congestion Avoidance
After ssthresh: additive increase (+1 MSS per RTT). Linear growth. Multiplicative decrease on loss.
Fast Retransmit
3 duplicate ACKs → retransmit without waiting for timeout. Faster than timeout-based recovery.
Fast Recovery
After 3 dup ACKs: halve cwnd, set ssthresh, avoid slow start. TCP Reno / CUBIC behavior.
